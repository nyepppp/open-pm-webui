# PM工作台重构计划

## Context

当前PM工作台已实现基础功能：项目创建、10个模块的条目管理、版本快照、追溯关系、PRD检查等。但存在以下核心问题：

1. **功能同质化**：所有10个模块（PRD/需求/参数/测试/风险/竞品/路线/会议/验收/FAQ）使用完全相同的表单（标题+优先级+文本内容），没有区分各模块的业务特性
2. **布局混乱**：所有模块平铺为标签页，没有按业务分类组织
3. **缺少富文本**：所有内容都是纯文本textarea，PRD等需要富文本编辑
4. **侧边栏缺失**：PM页面内部没有功能导航，版本/项目管理入口不明显
5. **Agent逻辑简单**：AI辅助只是生成提示词，没有智能分析能力

## Approach

### Phase 1: 模块分类与侧边栏重构

**1.1 重新定义模块分类**

按PRD业务领域将10个模块分为4大类：

| 分类 | 模块 | 编辑器类型 |
|------|------|-----------|
| **规划类** | PRD文档、需求池、竞品分析、路线图 | 富文本 / 结构化表单 |
| **设计类** | 参数清单、测试用例 | 结构化表单 |
| **执行类** | 风险问题、会议纪要 | 富文本 / 表单 |
| **复盘类** | 验收复盘、FAQ/培训 | 富文本 |

**1.2 PM页面内嵌侧边栏**

- 左侧边栏：项目选择 + 模块分类导航（可展开/折叠）
- 顶部：项目信息 + 版本切换器
- 主区域：根据选中模块显示对应的编辑器

**1.3 各模块编辑器设计**

| 模块 | 编辑器类型 | 字段设计 |
|------|-----------|---------|
| PRD文档 | 富文本编辑器 | 标题、版本、状态、富文本内容、附件 |
| 需求池 | 结构化表单 | 标题、优先级、状态、来源、用户角色、期望收益、关联模块 |
| 参数清单 | 结构化表单 | 模块、功能、字段名、类型、必填、默认值、枚举值、说明、来源PRD |
| 测试用例 | 结构化表单 | 标题、前置条件、操作步骤、预期结果、关联需求、优先级 |
| 风险问题 | 富文本+表单 | 标题、风险等级、概率、影响范围、负责人、应对方案、截止时间、富文本描述 |
| 竞品分析 | 富文本 | 竞品名称、场景、核心能力、差异点、可借鉴点、风险/机会 |
| 路线图 | 富文本 | 目标、范围、关键需求、依赖、计划时间、验收指标 |
| 会议纪要 | 富文本 | 时间、参与人、背景、结论、待办事项、风险 |
| 验收复盘 | 富文本 | 验收范围、结果、通过项、遗留问题、复盘结论、后续动作 |
| FAQ/培训 | 富文本 | 问题、回答、适用对象、关联功能、更新记录 |

### Phase 2: 富文本编辑器集成

- 调研Open WebUI现有富文本编辑器（如TipTap、Quill等）
- 为PRD文档、风险、竞品、路线、会议、验收、FAQ等模块集成富文本编辑器
- 支持Markdown、图片、表格等基础格式

### Phase 3: 版本与项目管理强化

- 顶部项目信息栏：项目名称、当前版本、版本切换下拉框
- 版本快照对比功能增强
- 项目设置入口（重命名、删除、导出）

### Phase 4: Agent智能辅助升级

- 基于当前模块内容智能生成建议
- PRD完整性自动检查（已有基础）
- 需求与测试用例的自动关联建议
- 风险识别与预警

## Key Files

- `src/routes/(app)/pm/+page.svelte` - 主页面重构
- `src/lib/components/pm/` - 新增PM专用组件目录
  - `PMModuleNav.svelte` - 模块分类导航
  - `PMProjectHeader.svelte` - 项目头部信息
  - `PMRichEditor.svelte` - 富文本编辑器封装
  - `PMFormEditor.svelte` - 结构化表单编辑器
  - `PMVersionSelector.svelte` - 版本选择器
- `src/lib/apis/pm/index.ts` - API扩展（如需新增字段）

## Risks & Open Questions

1. **富文本编辑器选型**：需要确认Open WebUI是否已有集成或依赖特定编辑器
2. **数据迁移**：现有数据都是`{text: string}`格式，新字段需要兼容旧数据
3. **性能**：富文本编辑器在大量条目时的性能表现
4. **后端扩展**：部分新字段可能需要后端model扩展，但优先前端兼容处理

## 实施优先级

1. **P0**: 模块分类侧边栏 + 项目头部重构（解决布局和分类问题）
2. **P1**: 各模块差异化表单（解决功能同质化）
3. **P2**: 富文本编辑器集成（PRD等模块需要）
4. **P3**: Agent智能辅助升级
