# 提交 PM 工具台并修复启动阻塞

## Context
- 当前仓库已完成 PM 工具台主体实现，涉及后端模型/路由/迁移、前端 `/pm` 页面、PM API client、用户菜单入口与后端 router 注册。
- PM 范围验证已通过：后端 Python 编译通过，PM 页面局部 `svelte-check` 通过 0 errors / 0 warnings。
- 仓库全量 `svelte-check` 当前存在大量非 PM 历史错误，已观察到集中在 `src/routes/auth/+page.svelte`、`src/routes/s/[id]/+page.svelte`、`src/routes/watch/+page.svelte` 等文件。
- 当前用户明确要求：先提交当前 PM 工具台改动；然后继续处理非 PM 历史错误；最后把项目跑起来并给出访问地址。

## Approach
1. **提交当前 PM 工具台改动**
   - 先检查 `git status --short`，区分应提交代码与不应提交的工具生成目录。
   - 纳入 PM 相关文件：
     - 后端 PM model/router/migration。
     - 后端 main router 注册。
     - 前端 PM API client、PM 页面、用户菜单入口。
   - 排除 `.orgii/` 工具目录和无关临时文件。
   - 运行提交前快速验证：PM 后端编译、PM 页面局部 Svelte 检查、`git diff --check`。
   - 执行本地 commit，提交信息建议为：`feat: add product manager workspace`。

2. **定位并修复全量 Svelte 检查中的非 PM 阻塞错误**
   - 重新运行直接入口方式的全量检查：`node node_modules\@sveltejs\kit\svelte-kit.js sync && node node_modules\svelte-check\bin\svelte-check --tsconfig .\tsconfig.json`。
   - 优先修复高频、集中、会阻塞开发启动信心的问题：
     - `$i18n` 被推断为 `unknown` / 非 store 的类型错误。
     - `src/routes/s/[id]/+page.svelte` 中 `chat`、`user`、`messages` 等隐式 any / nullable 类型错误。
     - `src/routes/watch/+page.svelte` 中 `videoId` 可能为 null 的编码错误。
   - 采用最小修复，不重构业务逻辑，不扩展功能；只补类型、空值保护或已有类型引用。
   - 每轮修复后重新跑全量 `svelte-check`，直到错误数量显著下降或全部通过；如存在非当前范围的海量历史问题，记录剩余错误文件和原因。

3. **验证 PM 工具台未回归**
   - 修复非 PM 错误后再次运行 PM 页面局部检查，确保 `/pm` 页面仍为 0 errors / 0 warnings。
   - 再次运行 PM 后端编译，确认后端新增文件未受影响。

4. **启动项目并提供访问地址**
   - 使用项目现有脚本启动前端开发服务，优先尝试 `npm run dev:5050`；如果端口冲突则改用 `npm run dev -- --port <可用端口>`。
   - 观察终端输出，确认 Vite/SvelteKit 服务实际监听的 URL。
   - 如后端也必须独立启动才能完整访问 PM API，则根据仓库现有启动方式启动或说明前端地址与后端依赖状态。
   - 最终向用户提供可访问地址，例如 `http://localhost:5050/pm`，并说明服务进程状态。

5. **修复提交策略**
   - PM 工具台作为第一笔提交。
   - 非 PM 历史错误修复建议作为第二笔提交，提交信息建议为：`fix: resolve existing svelte check blockers`。
   - 若只是为了本地启动做临时环境处理，不提交临时改动。

## Key Files
- `backend/open_webui/main.py`：确认 PM router 已注册。
- `backend/open_webui/models/pm.py`：PM 数据模型、Pydantic schema、内置模板、工作流、业务表操作。
- `backend/open_webui/routers/pm.py`：PM API 路由。
- `backend/open_webui/migrations/versions/f6a7b8c9d0e1_add_pm_workspace_tables.py`：PM 表结构迁移。
- `src/lib/apis/pm/index.ts`：前端 PM API client。
- `src/routes/(app)/pm/+page.svelte`：PM 工具台主页面。
- `src/lib/components/layout/Sidebar/UserMenu.svelte`：PM 工具台入口。
- `src/routes/auth/+page.svelte`：全量检查中 `$i18n` 类型错误的重点修复对象。
- `src/routes/s/[id]/+page.svelte`：全量检查中隐式 any / props 类型错误的重点修复对象。
- `src/routes/watch/+page.svelte`：全量检查中 nullable 参数错误的重点修复对象。

## Risks & Open Questions
- 全量 `svelte-check` 当前错误数量很大，可能包含多个历史模块的类型债；若全部修复范围过大，应优先处理启动和主流程阻塞错误，并明确剩余错误清单。
- 项目启动可能依赖后端环境变量、数据库迁移或 Python 服务；如果前端能启动但 PM API 报错，需要继续执行迁移/后端启动验证。
- `.orgii/` 是工具生成目录，不应提交。
