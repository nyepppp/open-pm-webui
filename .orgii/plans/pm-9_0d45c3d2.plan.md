# PM 模块溯源信息展示 + 9项缺陷修复

## Context

当前 PM 模块基于 SvelteKit + OpenWebUI 框架，使用统一 `ModuleEntry` 数据模型存储各模块条目。PRD 定义了完整的 Traceability（溯源）模块和 Versioning（版本）模块，但当前实现存在以下问题：

- **溯源信息不可见**：各模块条目页面看不到该条目的溯源关系（关联需求、来源文档、所属模块/功能等）
- **项目列表缺字段**：项目卡片不支持编辑名称/描述，不显示创建时间
- **工作台缺版本**：工作台缺少版本绑定和显示，各模块内容无法按版本关联
- **PRD 导入不完整**：只支持 md/txt，不支持 docx/doc
- **PRD 版本选择器错误**：PRD 编辑器内版本选择器是手动选择，应自动绑定工作台设定的当前版本
- **产品架构缺思维导图模式**：只有卡片列表，不支持思维导图视图
- **原型走查功能薄弱**：只支持编辑名称，不支持 UI 走查（图片上传、标注、评审记录）
- **路线图甘特图混乱**：使用自绘 SVG，布局错乱，需换开源甘特图组件；"同步到日程"应在每条路线图内操作
- **排期表格不可编辑**：各列不支持内联编辑，日期不支持同步到日程

**技术栈**：Svelte 5 (runes)、SvelteKit、Tailwind CSS、OpenWebUI 框架

## Approach

### 步骤 1：项目列表页增强 — 支持编辑名称/描述 + 显示创建时间

**修改文件**：`src/routes/(app)/pm/+page.svelte`

1. 在项目卡片上添加"编辑"按钮（hover 显示），点击后进入内联编辑模式
2. 内联编辑模式：name 变为 input，description 变为 textarea，增加保存/取消按钮
3. 调用已有的 `updateProject(token, projectId, { name, description })` API
4. 卡片上显示 `dayjs(project.createdAt).format('YYYY-MM-DD')` 创建时间
5. 添加 `updateProject` 的 import（已在 API 层存在但未在页面使用）

### 步骤 2：工作台增加版本绑定和展示

**修改文件**：`src/routes/(app)/pm/[projectId]/+page.svelte`

1. 在工作台顶部状态卡片区域增加"当前版本"显示卡片，读取 `currentVersion` store
2. 增加版本切换下拉（复用 `PMVersionSelector.svelte`），切换版本时更新 `currentVersion`
3. 在各模块分组卡片和条目上，显示条目所属版本标签（读取 entry 的 `versionId` 字段）
4. 在"最近更新"列表中，每条记录旁显示版本标签

### 步骤 3：各模块条目增加溯源信息面板

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. **表格视图**（requirement, parameter, testcase, roadmap, prototype, schedule）：在每行操作列增加"溯源"图标按钮，点击弹出溯源侧滑面板（Drawer）
2. **卡片/表单视图**（risk, competitor, faq, acceptance, meeting, product-architecture）：在卡片展开或详情编辑页增加"溯源"标签页/折叠区
3. **溯源面板内容**（根据 PRD §3.5 Traceability 模块定义）：
   - 来源信息：source（手动/Excel/AI）、sourceDocument（来源 PRD）、创建时间
   - 关联关系：requirementId（关联需求）、parameterId（关联参数）、featureName（所属功能）、moduleName（所属模块）
   - 版本信息：versionId（所属版本）
   - 跳转链接：点击关联 ID 可跳转到对应模块页面
4. **溯源数据来源**：从 entry.data / entry.metadata 中提取已有字段；版本从 entry.versionId 读取；同时调用 `getRelations` API 查询关联关系（如有）

### 步骤 4：PRD 导入支持 docx/doc

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. 引入 `mammoth` 库（轻量 docx → HTML 转换，~50KB）：`npm install mammoth`
2. 修改 `onMdFileSelected` 函数，当文件扩展名为 `.docx` 或 `.doc` 时，使用 `mammoth.convertToHtml({ arrayBuffer })` 将内容转为 HTML
3. 转换后的 HTML 按现有逻辑拆分为 PRDSection
4. 更新文件选择器的 `accept` 属性为 `.md,.markdown,.txt,.docx,.doc`
5. 移除 docx/pdf 导入的 toast 提示拦截

### 步骤 5：PRD 编辑器版本选择器改为自动绑定

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. 将 PRD 编辑器顶部的版本 `<select>` 从手动选择改为**只读显示**当前工作台版本
2. 新建条目时，自动将 `currentVersion` 的 ID 写入 entry 的 `versionId` 字段
3. 编辑已有条目时，显示条目创建时绑定的版本（只读标签），不可手动切换
4. 版本标签样式：`bg-blue-50 text-blue-600 rounded text-xs px-2 py-1`

### 步骤 6：产品架构增加思维导图模式

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. 产品架构模块卡片列表上方增加"视图切换"：卡片列表 / 思维导图
2. 思维导图模式复用已有的 `PMMindMap.svelte` 组件
3. 点击已有条目时，打开全屏思维导图编辑器（当前已有此逻辑，但仅自动打开第一个条目）
4. 思维导图视图下，条目列表改为只显示条目卡片，点击进入全屏 mindmap 编辑

### 步骤 7：原型走查增强 — 支持图片上传+标注+评审

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. 原型模块编辑器增加三个标签页：
   - **设计稿**：图片上传区域（支持 png/jpg/svg/webp），上传后显示缩略图列表，点击可全屏查看
   - **标注**：在设计稿图片上叠加标注层，支持点击添加标注点（x, y, text），标注以列表形式在侧边显示
   - **走查记录**：结构化表单 — 走查项名称、状态（通过/不通过/N/A）、问题描述
2. 原型编辑器的 `data` 字段存储 `annotations: [{id, x, y, text}]` 和 `checks: [{id, name, status, issue}]`
3. 创建原型条目时支持选择类型（图片/文件包/评审记录），并进入对应编辑模式

### 步骤 8：路线图甘特图替换为开源组件 + 同步到日程改进

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. 引入开源甘特图组件 `frappe-gantt`（轻量 SVG 甘特图，~5KB）：`npm install frappe-gantt`
2. 将自绘 SVG 甘特图替换为 frappe-gantt 组件渲染
3. 数据转换：将路线图 entries 转为 frappe-gantt 所需的 `Task[]` 格式（id, name, start, end, progress, dependencies）
4. "同步到日程"按钮从页面顶部移到每条路线图条目的操作菜单中（行内按钮）
5. 修复同步到日程功能：确保 `createCalendarEvent` 的参数正确，日历 ID 有效，时间戳格式正确

### 步骤 9：排期表格内联编辑 + 日期同步日程

**修改文件**：`src/routes/(app)/pm/[projectId]/[module]/+page.svelte`

1. 排期表格各列支持双击进入内联编辑模式：
   - title → input
   - assignee → input
   - startDate / endDate → date input
   - progress → number input (0-100)
   - isMilestone → checkbox
   - status → select（草稿/评审中/已批准/已归档）
2. 编辑完成后自动调用 `updateEntry` 保存
3. 每行操作列增加"同步到日程"按钮，点击将该任务的日期同步为日历事件
4. 甘特图视图也改为使用 frappe-gantt 组件（与路线图共用）

## Risks & Open Questions

1. **mammoth 库兼容性**：mammoth 是纯 JS 库，可在浏览器端运行，但 doc 格式（旧版 Word）不完全支持，仅 docx 可靠解析。doc 格式需提示用户转为 docx。
2. **frappe-gantt 集成**：frappe-gantt 是原生 JS 库，需通过 `onMount` + `bind:this` 方式在 Svelte 中集成，需注意响应式更新和销毁。
3. **溯源数据完整性**：当前条目的 data 字段中已有部分溯源字段（source, requirementId, parameterId, featureName, sourceDocument），但历史数据可能缺失。溯源面板需对空字段做容错处理。
4. **图片上传存储**：原型走查的图片上传依赖后端文件上传 API，需确认 OpenWebUI 的文件上传接口可用。若不可用，暂时使用 base64 内联存储（不推荐大图片）。
5. **版本绑定的数据一致性**：PRD 编辑器的版本 ID 当前存储在 entry.data.versionId 中，需确保与 entry.versionId 字段（如有）保持一致。
6. **甘特图交互**：frappe-gantt 支持拖拽调整日期，但当前 API 层的 updateEntry 可能无法精确更新单个 data 字段，需确保更新逻辑正确。
7. **工作量评估**：本计划涉及 9 项修改，主要集中在 `[module]/+page.svelte`（该文件已 1468 行），建议分批实施避免回归。
