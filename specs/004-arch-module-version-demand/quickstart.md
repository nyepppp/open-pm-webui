# Quickstart: 产品架构模块版本溯源与需求关联

**Feature**: 004-arch-module-version-demand
**Date**: 2026-07-08

## 前置条件

- Open WebUI 开发环境已启动
- 数据库迁移已完成（包含新字段 `create_version`, `version_record`, `demand_relation`, `is_deleted`, `deleted_at`）
- 工作台版本上下文已配置

## 验证场景

### 场景 1: 创建模块并验证版本自动绑定

**步骤**:
1. 打开产品架构模块的表格视图
2. 点击"新增模块"按钮
3. 填写模块信息：
   - 名称：用户管理
   - Key：user_management
   - 数据类型：object
   - 必填：是
   - 描述：管理用户注册、登录等功能
4. 点击保存

**预期结果**:
- 模块成功创建
- `create_version` 自动填充为当前工作台版本（如 "2.1.0"）
- `version_record` 自动生成一条记录：
  ```json
  {
    "version": "2.1.0",
    "operator": "当前用户",
    "time": "2026-07-08T10:30:00Z",
    "change_detail": "初始创建"
  }
  ```

### 场景 2: 编辑模块并验证版本履历追加

**步骤**:
1. 在表格视图中找到"用户管理"模块
2. 点击"编辑"按钮
3. 修改描述为：管理用户注册、登录、密码重置等功能
4. 点击保存

**预期结果**:
- 模块信息更新成功
- `version_record` 追加一条新记录：
  ```json
  {
    "version": "2.2.0",
    "operator": "当前用户",
    "time": "2026-07-08T11:00:00Z",
    "change_detail": "修改了描述字段"
  }
  ```

### 场景 3: 绑定需求并验证标签展示

**步骤**:
1. 在表格视图中点击某模块的"关联需求"单元格
2. 在弹窗中点击"新增需求"
3. 填写需求信息：
   - 需求编号：REQ-001
   - 需求名称：用户注册需求
   - 文档链接：https://docs.example.com/req-001
4. 点击保存

**预期结果**:
- 需求标签显示在"关联需求"列中
- 点击标签可跳转至文档链接
- 再次点击单元格可看到已绑定的需求

### 场景 4: 导出版本履历为 CSV

**步骤**:
1. 鼠标悬浮在某条目的"版本履历"单元格上
2. 等待浮窗弹出
3. 点击浮窗中的"导出"按钮
4. 选择保存位置

**预期结果**:
- 浏览器触发 CSV 文件下载
- CSV 内容包含所有版本变更记录的完整字段
- 文件名为 `{条目名称}_version_history.csv`

### 场景 5: 思维导图只读浏览

**步骤**:
1. 点击顶部 Tab 切换到"思维导图"视图
2. 查看模块-功能-参数的树形结构
3. 尝试拖拽节点
4. 点击某个模块节点

**预期结果**:
- 树形结构正确渲染
- 拖拽操作被禁止，节点位置不变
- 点击节点弹出详情弹窗，显示基础元信息、版本溯源、关联需求

### 场景 6: 软删除与数据留存

**步骤**:
1. 在表格视图中选择某模块
2. 点击"删除"按钮
3. 确认删除
4. 查看数据库中该条目的 `is_deleted` 和 `deleted_at` 字段

**预期结果**:
- 条目从表格视图中消失（默认不展示已删除）
- 数据库中 `is_deleted` = true
- `deleted_at` 填充删除时间
- `version_record` 和 `demand_relation` 完整保留

### 场景 7: 存量数据迁移验证

**步骤**:
1. 执行数据迁移脚本
2. 查看历史条目的 `create_version` 字段
3. 编辑一条历史条目

**预期结果**:
- 历史条目的 `create_version` = "1.0.0"
- `version_record` = []
- `demand_relation` = []
- 首次编辑后自动生成第一条版本变更记录

## 常见问题排查

### 问题 1: create_version 未自动填充

**排查步骤**:
1. 检查工作台版本上下文是否正确配置
2. 查看浏览器控制台是否有 API 错误
3. 确认数据库字段 `create_version` 是否存在

### 问题 2: 版本履历浮窗加载缓慢

**排查步骤**:
1. 检查 `version_record` 数组长度
2. 确认浮窗仅展示最近 20 条记录
3. 检查是否有大量历史数据需要分页

### 问题 3: CSV 导出失败

**排查步骤**:
1. 检查浏览器是否支持 Blob API
2. 查看控制台是否有 JavaScript 错误
3. 确认 `version_record` 数据格式正确

## 性能基准
