# Implementation Plan: Agent Workflow Designer & Architecture Fix

**Branch**: `[007-agent-workflow-designer]` | **Date**: 2026-07-11 | **Spec**: [specs/007-agent-workflow-designer/spec.md](spec.md)

**Input**: Feature specification from `/specs/007-agent-workflow-designer/spec.md`

## Summary

Build a visual workflow designer (inspired by Dify) for PM workspaces, enabling users to drag nodes, connect them, and define data transformations. Extend agent capabilities to read workspace data contextually and perform cross-module operations with human confirmation. Fix the existing product architecture diagram rendering.

## Technical Context

**Language/Version**: Python 3.11, SvelteKit

**Primary Dependencies**: FastAPI, SvelteKit, SQLAlchemy, Svelte Flow (or custom canvas)

**Storage**: SQLite (local) / PostgreSQL (production)

**Testing**: pytest (backend), Playwright (frontend)

**Target Platform**: Web (Open WebUI fork)

**Performance Goals**: Tool calls ≤3s, RAG ≤1s, module switch ≤500ms

**Constraints**: <200ms p95 for API calls, <100MB memory for workflow execution

**Scale/Scope**: 10k users, 1k workflows per project

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

**Constitution v1.2.1 Checklist:**
- [x] Principle I (Manual-First): Workflow designer works without AI — users can manually create, edit, and run workflows
- [x] Principle II (Module-Centric): Workflow nodes map to PM modules (requirements, PRD, test cases, etc.)
- [x] Principle III (Human-Confirmed): Agent write operations require confirmation (FR-006); dangerous ops always confirmed
- [x] Principle IV (Data Isolation): `project_id` scoping on all workflow entities; no cross-project access
- [x] Principle V (Version-Controlled): Workflow execution history retained for audit and replay
- [x] Principle VI (Skill-as-Generic-Module): All agent capabilities are `SkillContract` modules or `pm_*` Tools
- [x] Deterministic Pipeline Engine: Workflow engine is custom async engine, NOT Timbal replacement
- [x] Performance: Tool calls ≤3s, RAG ≤1s, module switch ≤500ms
