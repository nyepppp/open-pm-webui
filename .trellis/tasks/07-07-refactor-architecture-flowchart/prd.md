# 重构产品架构页面与修复流程图模块

## Goal

重构产品架构页面（`/pm/[projectId]/architecture`），将现有三标签页（思维导图/模块管理/参数表格）整合为两种展示形态：
1. **思维导图形态**：只读，从表格层级自动生成模块-功能架构图
2. **表格形态**：可编辑，模块下管理功能，功能下管理参数（输入/输出/配置），支持版本、溯源、关联需求文档

同时修复流程图模块（`/pm/[projectId]/flowchart`）的点击无响应和加载性能问题。

## Background

### 当前架构页面问题
- 现有三个独立标签页（思维导图、模块管理、参数表格），数据不联动
- 思维导图需要手动维护，与表格数据脱节
- 用户体验割裂，信息分散

### 当前流程图问题
- 使用 ReactFlow 组件，点击无响应
- 加载时间太长，性能差
- 缺少项目版本信息展示

## Requirements

### 产品架构页面重构

#### 展示形态1：思维导图（只读）
- **技术方案**：使用 ECharts Tree/Graph 替换现有 MindMapCanvas
- 从表格数据自动生成模块-功能层级结构
- 节点方式：模块 → 功能
- 节点信息：模块名称、功能名称
- 只读模式，不可编辑
- 点击节点可跳转到表格形态对应位置

#### 展示形态2：表格（可编辑）
- **UI 形态**：树形表格（类似 Excel 层级表格）
  - 模块作为一级行，可展开/收起
  - 功能作为二级行，嵌套在模块下
  - 参数作为三级行，嵌套在功能下
- **模块层级**：可新增、编辑、删除模块
- **功能层级**：在模块下新增、编辑、删除功能
- **参数层级**：在功能下新增参数，支持：
  - 参数类型：输入、输出、配置
  - 参数信息：名称、Key、数据类型、默认值、是否必填、描述
  - 版本信息：关联项目版本
  - 溯源信息：来源文档
  - 关联需求：多选关联需求文档
- 支持展开/收起层级
- 支持搜索和筛选

#### 数据联动
- 表格数据变更后，思维导图自动刷新
- 思维导图节点点击可定位到表格对应行
- 版本切换时，表格和思维导图同步更新

### 流程图模块修复

#### 技术方案
- **替换 ReactFlow 为 @antv/g6**：更轻量，性能更好，中文文档友好

#### 修复点击无响应
- 诊断并修复 ReactFlowCanvas 组件的点击事件问题
- 确保节点点击、拖拽、连线等交互正常

#### 优化加载性能
- 分析加载慢的根因（ReactFlow 库体积大）
- 使用 @antv/g6 替换，减少包体积
- 添加加载状态指示

#### 补充项目版本信息
- 在流程图页面顶部显示当前项目版本
- 版本切换时流程图数据同步更新

## Acceptance Criteria

### 产品架构页面
- [ ] 思维导图形态正确展示模块-功能层级结构
- [ ] 思维导图为只读模式，节点可点击定位到表格
- [ ] 表格形态支持模块的新增、编辑、删除
- [ ] 表格形态支持功能的新增、编辑、删除（在模块下）
- [ ] 表格形态支持参数的新增、编辑、删除（在功能下）
- [ ] 参数支持类型选择（输入/输出/配置）
- [ ] 参数支持版本、溯源、关联需求文档
- [ ] 表格数据变更后，思维导图自动刷新
- [ ] 版本切换时，两个形态同步更新

### 流程图模块
- [ ] 流程图页面点击节点有响应
- [ ] 流程图加载时间优化（目标 < 2秒）
- [ ] 流程图页面显示当前项目版本信息
- [ ] 版本切换时流程图数据同步更新

## Out of Scope

- 架构页面的 AI 辅助功能（保持现有 AI 按钮不变）
- 流程图的高级编辑功能（保持现有功能不变）
- 后端 API 变更（使用现有 API）
- 流程图模块修复（本次优先完成产品架构页面，流程图修复视后续安排）

## Decisions

| 决策 | 选择 | 原因 |
|------|------|------|
| 表格形态 UI | 树形表格 | 适合大量数据，与现有 ParameterTable 风格一致 |
| 思维导图组件 | ECharts Tree | 轻量、性能好、中文文档完善 |
| 流程图组件 | @antv/g6 | 更轻量，性能更好（本次暂不实施） |
| 实施策略 | 优先产品架构 | 降低风险，确保核心功能先完成 |
