# 修复架构图页面加载、UI风格和按钮文案问题

## Goal

修复 `/pm/{projectId}/architecture` 页面（产品架构页面）的三个问题：加载状态异常、UI风格不一致、按钮文案错误。

## Requirements

1. **加载状态修复**
   - 诊断并修复页面一直处于"加载中..."的问题
   - 确保数据加载完成后正确渲染内容
   - 添加适当的错误处理和重试机制

2. **UI风格统一**
   - Tab 切换按钮（"架构图"/"模块详情"）样式与项目设计系统保持一致
   - 使用项目中已有的按钮/Tab样式规范

3. **按钮文案修正**
   - 将"参数详情"改为"模块详情"
   - 确保模块详情 Tab 下能正确显示内容

## Acceptance Criteria

- [ ] 页面加载正常，数据正确渲染，不再出现无限加载
- [ ] Tab 按钮样式与项目设计系统保持一致
- [ ] "参数详情"按钮文案改为"模块详情"
- [ ] 模块详情 Tab 下能正确显示模块和功能的参数列表
- [ ] 无编译错误，无控制台报错

## Notes

- 相关文件：`src/routes/(app)/pm/[projectId]/architecture/+page.svelte`
- 需检查数据加载逻辑（`loadData`函数）和状态管理
- 需检查 `ParameterTable` 和 `ModuleFeatureTree` 组件的渲染逻辑
