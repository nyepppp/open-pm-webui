# 流程图溯源功能 - 产品需求文档 (PRD)

## 1. 背景与目标

### 1.1 现状
当前流程图功能（`PMFlowchartEditor.svelte`）仅支持创建基础表单（开始、结束、处理、判断、参数输入/输出节点），缺乏与项目实际模块、功能、参数的关联能力。流程图沦为静态图示，无法反映真实的业务逻辑和依赖关系。

### 1.2 目标
实现流程图节点的**溯源能力**，使每个节点能够绑定到具体的业务实体（模块/功能/参数），支持版本化管理，并提供思维导图式展示和表格导出功能。

## 2. 用户故事

- **作为**产品经理，**我希望**流程图节点能关联到具体的需求文档和模块，**以便**快速追溯业务逻辑的源头。
- **作为**开发人员，**我希望**看到流程图中每个节点绑定的功能和参数，**以便**理解实现细节。
- **作为**项目经理，**我希望**导出流程图为表格，**以便**进行评审和归档。

## 3. 功能需求

### 3.1 节点溯源绑定 (Traceability Binding)

每个流程图节点应支持绑定以下实体：

| 绑定类型 | 说明 | 数据来源 |
|---------|------|---------|
| 需求文档 (PRD) | 关联到 PRD 条目 | `prd` 模块 entries |
| 模块 (Module) | 关联到产品模块 | `architecture` / `module` entries |
| 功能 (Feature) | 关联到功能点 | `requirement` / `feature` entries |
| 参数 (Parameter) | 关联到输入/输出参数 | `parameter` 模块 entries |

绑定数据结构：
```typescript
interface NodeTraceability {
  // 绑定的实体类型
  entityType: 'prd' | 'module' | 'feature' | 'parameter' | 'none';
  // 绑定的实体ID
  entityId: string;
  // 实体名称（冗余存储，用于展示）
  entityName: string;
  // 版本绑定
  versionId?: string;
  versionNumber?: string;
  // 绑定时间
  boundAt: number;
  // 绑定人
  boundBy?: string;
}
```

### 3.2 版本绑定 (Version Binding)

- 节点绑定实体时，可选择绑定特定版本
- 支持查看节点的历史版本变更
- 版本变更时，高亮显示受影响的节点

### 3.3 思维导图展示 (Mind Map View)

- 提供切换按钮：流程图视图 ↔ 思维导图视图
- 思维导图中，节点按层级展开：
  - 根节点：流程图名称
  - 一级节点：模块分类
  - 二级节点：功能点
  - 叶子节点：参数
- 点击节点展开/收起侧边栏详情

### 3.4 侧边栏详情 (Sidebar Detail)

点击节点时，右侧展开侧边栏展示：
- 节点基本信息（名称、类型、描述）
- 绑定的实体信息（类型、名称、版本）
- 关联的参数列表（输入/输出）
- 关联的需求文档片段
- 版本历史记录

### 3.5 表格导出 (Export to Table)

支持导出流程图为表格形式（CSV/Excel），包含列：
- 节点ID
- 节点名称
- 节点类型
- 绑定实体类型
- 绑定实体名称
- 绑定版本
- 输入参数
- 输出参数
- 描述

## 4. 非功能需求

### 4.1 性能
- 思维导图渲染支持虚拟化，单图支持 500+ 节点流畅展示
- 节点绑定查询响应时间 < 200ms

### 4.2 兼容性
- 向后兼容现有流程图数据格式
- 未绑定实体的节点默认显示为普通节点

### 4.3 用户体验
- 拖拽绑定：支持从模块列表拖拽到节点进行绑定
- 批量绑定：支持多选节点批量绑定到同一实体
- 视觉反馈：绑定后节点显示实体类型图标和版本标签

## 5. 验收标准

### 5.1 功能验收
- [ ] 节点可以成功绑定到 PRD、模块、功能、参数
- [ ] 绑定版本后，版本变更能正确反映
- [ ] 思维导图视图能正确展示层级关系
- [ ] 点击节点能正确展示侧边栏详情
- [ ] 导出表格包含所有必要字段

### 5.2 性能验收
- [ ] 100 个节点的流程图加载时间 < 1s
- [ ] 导出 100 条记录 < 500ms

### 5.3 兼容性验收
- [ ] 现有流程图数据能正常加载和编辑
- [ ] 未绑定节点的流程图能正常使用

## 6. 风险与依赖

### 6.1 技术风险
- **风险**: `@xyflow/svelte` 版本升级可能导致 API 变更
- **缓解**: 使用稳定的 0.1.x 版本，封装节点组件降低耦合

### 6.2 依赖
- 后端需要提供实体查询 API（已有 `getEntries`）
- 版本管理模块需要支持版本查询（已有 `Version` 类型）
