# PRD: 架构可视化优化修复 — 性能与交互问题

## Goal

修复上一轮架构可视化优化中引入的性能问题和交互问题，提升用户体验。

## Issues Found

### 1. 思维导图布局问题
- **问题**: 思维导图应该是双边展示（左右对称），当前是单向的
- **影响**: 信息展示不够直观，空间利用率低
- **期望**: 根节点在中心，子节点向左右两侧展开

### 2. 页面尺寸问题
- **问题**: 思维导图页面太小，显示区域不足
- **影响**: 节点拥挤，难以查看完整信息
- **期望**: 全屏或更大显示区域，支持缩放

### 3. 缺少版本信息
- **问题**: 思维导图上没有显示版本信息
- **影响**: 用户无法区分不同版本的架构
- **期望**: 在思维导图上显示当前版本号

### 4. 模块卡片点击卡顿
- **问题**: 点击模块卡片时很卡
- **影响**: 用户体验差，交互不流畅
- **期望**: 流畅的点击响应

### 5. 多余的添加模块按钮
- **问题**: 卡片下方有 "+ 添加模块" 按钮，与顶部重复
- **影响**: 界面冗余
- **期望**: 移除重复的添加按钮

### 6. 模块卡片信息缺失
- **问题**: 卡片缺少版本跨度、溯源、操作按钮等信息
- **影响**: 信息不完整，用户无法获取完整上下文
- **期望**: 补充完整信息展示

## Acceptance Criteria

### 思维导图
- [ ] 支持双边展示（左右对称布局）
- [ ] 页面显示区域更大，支持全屏
- [ ] 显示当前版本信息

### 模块卡片
- [ ] 点击响应流畅，无卡顿
- [ ] 移除卡片下方的 "+ 添加模块" 按钮
- [ ] 补充版本跨度、溯源信息
- [ ] 添加操作按钮（编辑、删除等）

## Notes

- 保持现有数据流不变
- 支持暗黑模式
- 响应式设计
