# Design: 架构可视化优化修复

## 1. 思维导图双边展示

### 布局算法
- 使用 d3.tree() 的 radial 布局或自定义 bilateral 布局
- 根节点在中心
- 第一层子节点（模块）向左右两侧展开
- 第二层子节点（功能）继续向外展开

### 实现方案
```typescript
// 使用 d3.tree() 的 size 设置为 [height, width]
// 自定义布局：将节点分为左右两组
// 左组：x 坐标为负，右组：x 坐标为正
```

### 页面尺寸
- 容器高度：100vh - header 高度
- 支持全屏模式
- 默认缩放级别：fit to screen

## 2. 版本信息显示

### 显示位置
- 思维导图右上角显示版本标签
- 格式：版本 v1.0.0

### 样式
- 小标签样式
- 背景色：蓝色
- 文字：白色

## 3. 模块卡片性能优化

### 卡顿原因分析
- 可能原因：频繁的 DOM 操作、复杂的计算、不必要的事件监听

### 优化方案
- 使用 `requestAnimationFrame` 优化动画
- 减少不必要的响应式更新
- 使用 CSS 动画代替 JS 动画
- 事件委托减少监听器数量

## 4. 模块卡片信息补充

### 新增信息
- 版本跨度：显示该模块涉及的版本范围
- 溯源信息：数据来源（自动/手动）
- 操作按钮：编辑、删除、查看详情

### 布局调整
- 卡片头部：模块名称 + 来源标签
- 卡片主体：描述 + 版本信息
- 卡片底部：操作按钮
