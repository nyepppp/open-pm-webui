# 流程图功能问题排查与修复

## Goal

系统性排查并修复流程图页面（`/pm/{id}/flowchart`）中的多个功能缺陷，确保形状渲染、Excalidraw 集成和实体绑定功能正常工作。

## Requirements

### 1. 画布形状渲染修复
- **问题**：无论选择何种形状（矩形/圆角/圆形/菱形/椭圆），创建的元素始终显示为矩形
- **期望**：根据用户选择的形状类型正确渲染对应的图形
- **涉及组件**：Excalidraw 画布、形状选择器

### 2. Excalidraw 集成修复
- **问题**：右侧面板（节点配置/溯源绑定）未融入 Excalidraw 工作流程
- **期望**：右侧面板应作为 Excalidraw 的扩展面板或工具栏的一部分，而非独立浮层
- **涉及组件**：右侧属性面板、Excalidraw 容器

### 3. 实体绑定按钮修复
- **问题**：PRD/模块/功能/参数按钮点击后无响应或查询不到相关实体
- **期望**：
  - PRD 按钮：应能查询并绑定到当前项目的 PRD 文档
  - 模块按钮：应能查询并绑定到当前项目的模块实体
  - 功能按钮：应能查询并绑定到当前项目的功能实体
  - 参数按钮：应能查询并绑定到当前项目的参数实体
- **涉及组件**：实体绑定按钮组、API 查询逻辑

## Acceptance Criteria

- [ ] 画布中创建的形状与选择的形状类型一致（矩形/圆角/圆形/菱形/椭圆）
- [ ] 右侧面板正确集成到 Excalidraw 工作流中，不再是独立浮层
- [ ] PRD 按钮点击后能正确查询并显示可绑定的 PRD 列表
- [ ] 模块按钮点击后能正确查询并显示可绑定的模块列表
- [ ] 功能按钮点击后能正确查询并显示可绑定的功能列表
- [ ] 参数按钮点击后能正确查询并显示可绑定的参数列表
- [ ] 所有修复不破坏现有流程图的其他功能

## Notes

- 项目使用 Excalidraw 作为流程图画布引擎
- 实体数据存储在项目管理系统中，需要通过 API 查询
- 右侧面板当前为独立组件，需要与 Excalidraw 容器融合
