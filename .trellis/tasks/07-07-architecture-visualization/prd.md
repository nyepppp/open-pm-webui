# PRD: 架构可视化优化 — 思维导图与卡片式模块列表

## Goal

优化产品架构页面的可视化展示，包括：
1. 将 Excalidraw 思维导图替换为更美观的 mindmap 组件
2. 将模块/功能管理页面从表格改为卡片式布局

## Requirements

### 1. 思维导图优化
- **问题**: 当前使用 Excalidraw 的连线样式丑陋
- **方案**: 替换为专门的 mindmap 组件，提供更美观的树形布局
- **参考**: 类似 [d3-mindmap](https://observablehq.com/@d3/mindmap) 或 [markmap](https://markmap.js.org/) 的视觉效果

### 2. 模块列表卡片化
- **问题**: 当前使用表格展示模块/功能，层级关系不清晰
- **方案**: 改为卡片式布局，层层递进：
  - 第一层：模块卡片（显示模块名称、来源、功能数量）
  - 第二层：展开后显示功能卡片列表
  - 第三层：最深处显示参数表格
- **交互**: 支持展开/收起、添加、删除、编辑描述

## Acceptance Criteria

### 思维导图
- [ ] 替换 Excalidraw 为专门的 mindmap 组件
- [ ] 支持树形布局，连线美观
- [ ] 支持节点点击交互
- [ ] 保持现有数据格式兼容

### 模块卡片
- [ ] 模块以卡片形式展示
- [ ] 卡片包含：模块名称、来源标签、功能数量、描述
- [ ] 点击卡片展开功能列表
- [ ] 功能以子卡片形式展示
- [ ] 最深处显示参数表格
- [ ] 保持现有增删改查功能

## Notes

- 使用 Svelte 5 + TypeScript
- 保持现有数据流（architectureStore、architectureService）
- 支持暗黑模式
- 响应式设计
