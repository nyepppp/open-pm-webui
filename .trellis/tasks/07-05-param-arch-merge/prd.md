# 参数配置与产品架构融合 + 思维导图视图

## Goal

将"参数配置"模块与"产品架构"模块融合为一个页面，支持切换视图。一级为模块，二级为功能，三级为参数。产品架构视图展示方式为思维导图。

## Context

- **当前实现**: `parameter` 和 `product-architecture` 是两个独立模块，各自有独立的路由和编辑器。
  - `parameter`: table 编辑器，含 moduleName/featureName 字段
  - `product-architecture`: mindmap 编辑器（`PMMindMap`）
- **用户期望**: 两者融合，参数按模块→功能→参数的三级树结构组织，产品架构以思维导图展示。

## Requirements

1. 参数配置页面增加视图切换：表格视图 + 思维导图视图
2. 思维导图视图展示三级结构：模块(一级) → 功能(二级) → 参数(三级)
3. 思维导图中点击参数节点可展开/编辑参数详情
4. 产品架构模块的思维导图保持现有功能不变
5. 表格视图保持现有的参数列表展示

## Acceptance Criteria

- [ ] 参数配置页面有视图切换按钮（表格/思维导图）
- [ ] 思维导图展示模块→功能→参数三级结构
- [ ] 思维导图可交互（展开/折叠节点）
- [ ] 表格视图功能不受影响
- [ ] 产品架构模块的思维导图独立功能不受影响

## Notes

- 这是一个新功能需求，不是 bug 修复
- 复杂度较高，需要 design.md 和 implement.md
