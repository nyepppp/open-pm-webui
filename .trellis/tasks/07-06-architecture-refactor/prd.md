# 重构产品架构模块-功能-参数层级展示

## Goal

重构产品架构页面，将"产品架构"和"参数配置"两个模块深度结合，以"模块-功能-参数"的层级概念展示产品架构，并支持版本信息展示和需求文档溯源功能。

## Background

当前产品架构页面 (`/pm/[projectId]/architecture`) 采用双标签页设计：
- **思维导图 (mindmap)**：以可视化方式展示模块-功能的树形结构
- **参数配置 (params)**：以表格形式展示参数，左侧有模块-功能导航树

这种设计导致模块-功能信息在两个标签页中重复展示，用户体验割裂。参数配置模块虽然支持参数管理，但缺乏与产品架构的深度融合。

## Requirements

### 1. 模块-功能-参数层级展示

**1.1 统一展示层级**
- 以"模块 → 功能 → 参数"的三级层级结构展示产品架构
- 左侧保留模块-功能导航树，点击功能后右侧展示该功能下的参数列表
- 参数列表需要展示：参数名、Key、类型（输入/输出/配置）、数据类型、默认值、必填、说明、状态

**1.2 模块和功能信息展示**
- 产品架构页面需要展示所有模块和功能信息
- 每个模块/功能需要显示版本信息（当前版本号）
- 支持查看模块/功能的详细信息和变更历史

**1.3 参数配置整合**
- 将现有的参数配置功能整合到产品架构页面中
- 支持在功能节点下直接添加、编辑、删除参数
- 参数表单需要包含：参数名、Key、所属模块、所属功能、参数类型、数据类型、默认值、必填、说明

### 2. 版本信息展示

**2.1 版本选择器**
- 在产品架构页面顶部添加版本选择器
- 支持选择不同版本查看历史架构
- 显示当前版本号和版本标签（里程碑/发布/评审）

**2.2 版本对比**
- 支持对比两个版本的架构差异
- 高亮显示新增、修改、删除的模块/功能/参数

### 3. 需求文档溯源

**3.1 参数溯源**
- 每个参数可以关联到对应的需求文档（PRD/requirement）
- 在参数列表中显示关联的需求文档信息
- 支持点击跳转查看需求详情

**3.2 多选支持**
- 支持为一个参数关联多个需求文档
- 支持批量选择参数并关联需求文档
- 支持在参数编辑时选择/取消关联需求文档

**3.3 溯源展示**
- 在参数详情中展示完整的需求文档溯源链
- 支持查看参数与需求文档的关联关系图

### 4. 交互优化

**4.1 树形导航**
- 左侧模块-功能树支持展开/折叠
- 支持搜索模块和功能
- 选中功能后高亮显示，右侧展示对应参数

**4.2 参数操作**
- 支持在参数列表中直接编辑参数
- 支持批量操作（批量删除、批量修改状态）
- 支持参数排序和筛选

**4.3 新建流程**
- 支持从模块层级开始新建功能
- 支持从功能层级直接添加参数
- 新建时需要选择关联的需求文档（可选）

## Acceptance Criteria

- [ ] 产品架构页面以"模块-功能-参数"层级结构展示
- [ ] 左侧导航树展示所有模块和功能，点击功能后右侧展示参数
- [ ] 参数列表包含完整的参数信息和操作按钮
- [ ] 版本选择器可以切换不同版本
- [ ] 当前版本号在页面中清晰可见
- [ ] 参数可以关联到需求文档（支持多选）
- [ ] 关联的需求文档在参数列表中可见
- [ ] 支持从参数跳转到关联的需求文档
- [ ] 模块/功能的增删改操作正常工作
- [ ] 参数的增删改操作正常工作
- [ ] 页面响应式适配，在移动端可用
- [ ] 暗黑模式支持
